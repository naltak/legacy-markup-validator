Document types support

Below is a list of sample documents for a number of document types the validator is supposed to be supporting. The first validate link will attempt validation with the current instance, for test purposes. The v.w.o instance can be taken as reference/comparison.

  • validate - with v.w.o - view: HTML 2.0
  • validate - with v.w.o - view: HTML 2.0 Strict
  • validate - with v.w.o - view: HTML 4.0 Strict
  • validate - with v.w.o - view: HTML 4.0 Transitional
  • validate - with v.w.o - view: HTML 4.0 Frameset
  • validate - with v.w.o - view: HTML 4.01 Transitional - (from the HTML 4.01 PR)
  • validate - with v.w.o - view: ISO-HTML (aka ISO/IEC 15445:2000)
  • validate - with v.w.o - view: XHTML 1.0 Strict
    validate - with v.w.o - view: XHTML 1.0 Strict (from XHTML Spec)
  • validate - with v.w.o - view: XHTML Basic 1.0
  • validate - with v.w.o - view: XHTML 1.1
  • validate - with v.w.o - view: XHTML/MathML2 (with FPI)
  • validate - with v.w.o - view: MathML 2.0 (minimal test document)
  • validate - with v.w.o - view: XHTML 1.1 served as application/xhtml+xml
  • validate - with v.w.o - view: SMIL 1.0 minimal document
  • validate - with v.w.o - view: SMIL 2.0
  • validate - with v.w.o - view: XHTML + MathML + SVG (from the 9 August 2002 WD)

User Interface tests

The following tests do not test the binary "validity" results, but the proper handling of options, errors, various outputs, etc. "validate" links use this instance, with validator.w3.org links added for reference comparison

  • validate - with v.w.o - /check/referer (deprecated)
  • validate - with v.w.o - /check/referer;verbose=1 (unsupported)
  • validate - with v.w.o - /check/referer;verbose=1;No200=1 (unsupported)
  • validate - with v.w.o - /check?uri=referer (supported, should validate this test collection)
  • validate - with v.w.o - /check?uri=referer;verbose=1 (supported, should validate this test collection)
  • validate - with v.w.o - /check?uri=refererr&verbose=1 (supported, should validate this test collection)
  • validate - with v.w.o - /check?uri=referer;verbose=1;No200=1 (supported, should validate this test collection)
  • validate - with v.w.o - Directory Redirect (should mention the redirect and validate the redirection target
  • validate - with v.w.o - script invoked with no parameters (gives an error)
  • validate - with v.w.o - uri parameter is empty
  • validate - with v.w.o - uri is missing http:// (silent fix)
  • validate - with v.w.o - 404 not found
  • validate - with v.w.o - 404 not found, with "validate error pages" ON
  • validate - with v.w.o - text/plain resource (refuses to validate)
  • validate - with v.w.o - httpd on a non-standard port (should be working transparently if :8000 server is running)

Bugs and Regression Tests

These should be tied to a bug report on either mailing-list and/or bugzilla whenever possible.

  • validate - with v.w.o - view: Bug 798: Ampersand as data in XHTML (also good as a test for warnings in otherwise valid result)
  • Long Lines (cause a core)
  • A trailing ASCII NUL character (trips up HTML::Parser (not really))
  • Undeclared namespace prefixes (colons ":" in element names (trips up parsing of onsgmls output))
